# Product Brief

This workspace simulates a small production Python service.

The agent's job is to:

- inspect the repository safely
- identify likely implementation hotspots
- produce a delivery report
- produce a patch plan
- persist a useful task journal for follow-up questions

The goal is not to pretend the agent is magically market-ready.
The goal is to demonstrate a production-style architecture:

- explicit task packet assembly
- strong execution boundaries
- persistent coding memory
- local follow-up semantics
- structured delivery artifacts
